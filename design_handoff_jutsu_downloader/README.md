# Handoff: Jutsu Downloader (десктопное приложение)

## Overview
«Jutsu Downloader» — десктопное приложение (в первую очередь macOS, также Windows/Linux):
одновременно **мини-браузер одного сайта** (внутри открывается ТОЛЬКО jut.su) и **качалка
аниме**. Пользователь смотрит аниме внутри приложения или скачивает серии в выбранном
качестве для офлайна. Под капотом уже есть готовая качалка: заходит на страницу серии по
кукам премиум-аккаунта пользователя, достаёт прямые ссылки на mp4 и качает файл.
Интерфейс — на русском.

Прототип покрывает 3 главных экрана + ключевые состояния + style-guide, и поддерживает
переключение светлой/тёмной темы.

## About the Design Files
Файлы в этом бандле — **дизайн-референс, сделанный в HTML** (прототип, показывающий
задуманный вид и поведение), а НЕ продакшн-код для копирования. Задача — **воссоздать этот
дизайн в среде вашего кодовой базы** (Electron + React/Vue, Tauri, SwiftUI, нативный код и
т.д.), используя её установленные паттерны и библиотеки. Если среды ещё нет — выбрать
подходящий фреймворк. Для десктопного приложения с webview логичен **Electron** или
**Tauri**: системный webview показывает jut.su, нативный слой перехватывает навигацию
(ограничение доменом jut.su), читает куки сессии и качает mp4.

`Jutsu Downloader.dc.html` — самодостаточный прототип (React-логика внутри). `support.js` —
рантайм-шелл прототипа, для продакшена не нужен, приложен только чтобы прототип открывался.

## Fidelity
**High-fidelity.** Финальные цвета, типографика, отступы, радиусы, тени и интерактив.
Воссоздавайте UI пиксель-в-пиксель средствами вашей кодовой базы. Единственное упрощение —
обложки аниме нарисованы CSS-градиентами-плейсхолдерами (в проде — реальные постеры с
jut.su), а встроенный webview в прототипе имитирован статичной вёрсткой jut.su (в проде —
настоящий webview-компонент).

Размер окна прототипа: **1280 × 800** (контент), титлбар macOS сверху ~38px.

## Screens / Views

### Общая оболочка окна
- Титлбар (38px): 3 traffic-lights слева, по центру «Jutsu Downloader».
- Плавающая круглая **FAB** (60px, accent) правый-нижний угол — только на Экране 1.
- Тосты — по центру снизу окна. Модалка ошибки — оверлей поверх всего.
- В прототипе есть внешний «навигатор» (переключение экранов/состояний + кнопка темы) — это
  инструмент прототипа, в проде не нужен.

### Экран 1 — «Просмотр» (встроенный jut.su)
- **Purpose**: смотреть аниме внутри приложения; инициировать скачивание текущей серии.
- **Layout**: колонка — тонкий верхний бар (52px) + область webview (flex, скролл).
- **Верхний бар**: кнопки ← (активна) → (disabled в примере) ⌂ «Домой»; по центру адресный
  блок с зелёной точкой, «jut.su» и названием текущей страницы; справа пилюля «Премиум
  активен» (зелёная точка + текст `#34D399`) и шестерёнка ⚙. Навигация ← → ограничена
  доменом jut.su.
- **Webview**: реальный сайт jut.su. Два варианта страницы:
  - **(а) Страница серии** — хлебные крошки, видеоплеер 16:9 (radius 12, тёмный, круглая
    accent-кнопка play, скраббер + время + «1080p·HD»), заголовок серии, кнопки Пред./След.,
    сетка миниатюр серий сезона (текущая серия — accent-бордер).
  - **(б) Главная страница аниме** — обложка 196×280, заголовок, чипы жанров, описание,
    кнопка «▶ Смотреть 1 серию», и **баннер-подсказка** «Открыть список серий →» (accent-тинт,
    ведёт на Экран 3).
- **FAB**: accent-круг со стрелкой загрузки ⬇, бейдж-счётчик активных загрузок (красный
  `#FF5C5C`, белый текст, ободок цвета окна). Клик → поповер-меню (открывается вверх, 344px):
  1. **«⬇ Скачать эту серию»** — активна ТОЛЬКО на странице серии: показывает распознанные
     аниме·сезон·серию, дропдаун качества (1080p·Full HD / 720p / 480p / 360p), кнопка
     «Добавить в очередь». На странице аниме — задизейблена с подсказкой открыть список серий.
  2. **«☰ В библиотеку»** → Экран 2.

### Экран 2 — «Библиотека» (скачанное + очередь)
- **Purpose**: управлять скачанным и очередью загрузок.
- **Тулбар (56px)**: ← назад к просмотру, заголовок «Библиотека», поиск, сортировка
  (дропдаун «Недавние»), индикатор диска (мини progress + «87 ГБ»), выбор папки
  «📁 ~/Downloads/Anime».
- **Активные загрузки** (если есть): заголовок + счётчик; строки-карточки: мини-обложка 44px,
  тайтл + «Сезон·Серия · качество», справа %/«Пауза», «done/total МБ · скорость МБ/с»,
  кнопки пауза(❚❚)/возобновить(▶) и отмена(✕), снизу прогресс-бар (accent, `transition:width`).
- **Скачано**: адаптивная сетка карточек (`minmax(196px,1fr)`), обложка 2:2.7 с названием и
  бейджем сезонов, под ней тайтл и «N серий · X ГБ».
- **Раскрытие серий** (клик по карточке): панель со списком серий — № + название,
  длительность, бейдж качества, «✓ скачано», кнопки «▶ смотреть» (accent) и «🗑 удалить».
- **Пустое состояние**: иконка, «Пока ничего не скачано», подсказка + кнопка «Перейти к
  просмотру».

### Экран 3 — «Список серий / пакетная загрузка» (парсер страницы аниме)
- **Purpose**: распарсить страницу аниме (напр. jut.su/faairytail/), вытащить все серии,
  качать пачкой.
- **Тулбар (52px)**: ← , «Список серий», справа «☰ В библиотеку».
- **Шапка**: обложка 150×214, заголовок, «Сезонов: 4 · Серий: 353», чипы жанров, кнопка
  «▶ Смотреть» (→ Экран 1).
- **Список по сезонам**: заголовок сезона + «N серий» + «Выбрать сезон»; строки: чекбокс
  (accent при выборе), «Серия N — <название>», длительность, статус-бейдж
  (не скачано / в очереди / ✓ скачано) с соответствующими тинтами.
- **Панель массовых действий (закреплена снизу, 72px, glass-фон + blur)**: «Выбрать всё /
  Снять всё», дропдаун общего качества, счётчик «Выбрано серий: N · качество», крупная кнопка
  «⬇ Скачать выбранное (N)» (accent; задизейблена при N=0) → добавляет в очередь и ведёт на
  Экран 2.
- **Skeleton**: при загрузке списка — шиммер-плейсхолдеры шапки и строк
  (`@keyframes` сдвиг градиента 1.4s).

## Interactions & Behavior
- **Навигация между экранами**: FAB-меню и «В библиотеку» с Экрана 1; «Смотреть»/«В
  библиотеку» с Экрана 3; ← возврат к просмотру из Библиотеки; баннер/подсказка → Экран 3.
- **Скачивание одной серии**: FAB → «Скачать эту серию» → выбор качества → «Добавить в
  очередь» → тост «Добавлено в очередь…» → переход в Библиотеку. Реально: читать куки
  сессии, распарсить страницу серии, достать mp4-URL нужного качества, поставить в очередь.
- **Пакетное скачивание**: отметить серии/сезоны → выбрать качество → «Скачать выбранное (N)»
  → тост → Библиотека.
- **Очередь**: прогресс тикает; пауза/возобновление; отмена (тост «Загрузка отменена»); по
  завершении — тост «Загрузка завершена: …», серия исчезает из активных и появляется в
  скачанном.
- **Переключение темы**: мгновенное, через `data-theme` на корне (все токены — CSS-переменные).
- **Анимации**: меню/дропдауны `jd-menu` .14–.16s ease (fade + translateY + scale); тосты
  `jd-toast` .25s (fade + translateY), автоскрытие ~3.2s; skeleton `jd-shimmer` 1.4s linear;
  прогресс-бар `transition: width .5s linear`.
- **Hover**: строки списков подсвечиваются (`#181820`/light-эквивалент); кнопки — стандартные
  hover вашего дизайн-слоя.

## State Management
- `theme`: 'dark' | 'light'
- `screen`: 'watch' | 'library' | 'episodes'
- `watchVariant`: 'episode' | 'anime' (какая страница jut.su открыта)
- `fabOpen`, `qualityOpen`, `batchQualityOpen`: открытость поповера/дропдаунов
- `quality`, `batchQuality`: выбранное качество (одиночное / пакетное)
- `expandedAnime`: id раскрытого аниме в Библиотеке (или null)
- `libraryEmpty`, `episodesLoading`, `errorOverlay`: флаги состояний
- `selected`: { [`s{season}-{index}`]: bool } — выбор серий на Экране 3
- `downloads`: [{ id, animeId, title, sub, quality, done, total, speed, paused, active }]
- `toasts`: [{ id, icon, iconBg, border, text }] (автоудаление по таймеру)

**Data fetching (прод)**: список аниме/серий в библиотеке из локальной БД; парсер страницы
аниме для Экрана 3; менеджер загрузок с реальным прогрессом; проверка валидности сессии
(куки jut.su).

## Error / Loading States
- **Skeleton** — загрузка списка серий (Экран 3).
- **Ошибка «Сессия истекла»** — модалка-оверлей: иконка ⚠, «Сессия истекла», пояснение про
  недействительные куки, кнопки «Позже» и «Обновить вход» (переавторизация на jut.su).
- **Тосты** — успех (зелёный тинт, ✓), инфо/завершение (accent-тинт, ⬇), ошибка/отмена
  (красный тинт).
- **Пустая библиотека** — см. Экран 2.

## Assets
- **Шрифт**: Inter (Google Fonts) — замените на локально-бандленый Inter в проде.
- **Обложки аниме**: в прототипе — CSS-градиенты-плейсхолдеры; в проде тянуть реальные
  постеры с jut.su.
- **Иконки**: в прототипе Unicode-глифы (⬇ ☰ ⚙ ⌂ ▶ ✓ 🗑 ⚠ ‹ › ▾ ✕). В проде используйте
  вашу icon-библиотеку.
- Бренд jut.su (логотип «jut.su») — принадлежит сайту; используйте по факту его отображения
  в webview.
